# Hanzi Hardsub Reader

Hanzi Hardsub Reader is a Manifest V3 Chrome extension that extracts hard-coded Chinese subtitles from YouTube video frames, then presents recognized text with optional pronunciation (Pinyin/Zhuyin) and English translation.

## What it does

- Runs OCR locally in an offscreen extension document using PaddleOCR + ONNX Runtime Web.
- Lets you draw and save subtitle-region presets directly on the YouTube player.
- Streams recognized text into an in-page side panel with timestamped entries.
- Supports phonetic annotation toggles (Pinyin and Zhuyin).
- Supports two translation providers:
  - **Local**: Chrome Built-in AI Translation API (when available).
  - **Web**: Google Translate web endpoint (`translate.googleapis.com`).

## Browser compatibility

### Core extension support

- **Required browser**: Google Chrome **120+**.
- Basis for this requirement:
  - Build target is `chrome120` (`esbuild.config.js`).
  - Uses Manifest V3 APIs including `chrome.offscreen`, `chrome.scripting`, and service workers.

### Translation provider compatibility

- **Local translation (Chrome Built-in AI)**:
  - Depends on Translation API availability in your Chrome build.
  - May require enabling `chrome://flags/#translation-api`.
  - May require language pack/model availability for `zh -> en`.
- **Web translation (Google Translate endpoint)**:
  - Requires network access to `https://translate.googleapis.com`.
  - Useful fallback when Built-in AI translation is unavailable.

### Tested version

- Verified in: **Chrome 147.0.7727.138 (Official Build) (arm64)**.

## Local setup (load unpacked extension)

1. Clone the repository.
2. Install dependencies:
   ```bash
   npm install
   ```
3. Build extension bundles:
   ```bash
   npm run build
   ```
4. Open `chrome://extensions`.
5. Enable **Developer mode**.
6. Click **Load unpacked** and select the repository's `extension/` directory.

## Using the extension

1. Open a YouTube video page (`https://www.youtube.com/...`).
2. Open the extension popup and click **Draw New Subtitle Area**.
3. Draw or adjust the subtitle capture region.
4. Start recognition (from popup or side panel controls).
5. Expand entries in the side panel to request translation.
6. Use side panel settings to enable Pinyin/Zhuyin and choose translation provider.

## Development

### Scripts

- `npm run build` — Build popup/content/offscreen bundles and copy ONNX Runtime WASM assets.
- `npm run watch` — Rebuild on file change.
- `npm run lint` — Run ESLint over `extension/**/*.js`.
- `node tests/e2e-extension.js` — Run Playwright-based end-to-end checks.

### Devcontainer

This repository includes `.devcontainer/devcontainer.json` for VS Code / Codespaces.

1. Open the repository in VS Code.
2. Run **Dev Containers: Reopen in Container**.
3. Wait for `postCreateCommand` (`npm install`) to finish.
4. Run normal project scripts inside the container terminal.

## Troubleshooting

- **Popup says “Open YouTube to use”**: run only on `https://www.youtube.com/*` pages.
- **Local translation fails**: verify Translation API support in your Chrome build and flag state.
- **Web translation fails**: check network access to `translate.googleapis.com`.
- **No OCR text detected**: redraw a tighter subtitle box and ensure subtitles are visible in-frame.
