  <action>
Create `extension/content/overlay.js` as a module that exports a `SelectionOverlay` class (or object) with the following API:

**Exported interface:**
```javascript
export class SelectionOverlay {
  constructor() // finds video element, creates overlay container
  activateDrawMode() // enables crosshair cursor, starts listening for draw
  deactivateDrawMode() // returns to normal cursor, stops draw listeners
  getSelectionRect() // returns {x, y, width, height} in CSS pixels relative to video, or null
  getVideoIntrinsicRect() // returns {x, y, width, height} in video intrinsic pixels (for canvas capture)
  hasSelection() // returns boolean
  destroy() // removes all DOM elements and listeners
}
```

**DOM structure created by the overlay:**
```
div#ycr-overlay-container (positioned over the YouTube video player)
  div#ycr-selection-box (the drawn rectangle)
    div.ycr-handle.ycr-handle-nw (top-left corner)
    div.ycr-handle.ycr-handle-ne (top-right corner)
    div.ycr-handle.ycr-handle-sw (bottom-left corner)
    div.ycr-handle.ycr-handle-se (bottom-right corner)
```

**Finding the video element:**
- Use `document.querySelector('#movie_player video')` to find the YouTube video element.
- If not found immediately, use a MutationObserver on `document.body` watching for `childList` and `subtree` changes. When a `video` element appears inside `#movie_player`, resolve. Timeout after 5 seconds and log `[YCR] Video element not found`.
- Store a reference to the video element for coordinate conversion.

**Overlay container positioning:**
- The overlay container (`#ycr-overlay-container`) must be positioned as a child of the `#movie_player` element (or its parent if needed).
- `position: absolute; top: 0; left: 0; width: 100%; height: 100%;`
- `z-index: 2147483646` (above YouTube player UI)
- `pointer-events: none` by default (so YouTube controls still work)
- When in draw mode: `pointer-events: auto; cursor: crosshair;`

**Drawing interaction (per D-03):**
- When `activateDrawMode()` is called:
  - Set overlay container to `pointer-events: auto; cursor: crosshair;`
  - On `mousedown` on the overlay container: record start position (clientX/clientY relative to overlay).
  - On `mousemove` on `document`: update the selection box dimensions. Show the box with:
    - `border: 2px solid rgba(255, 255, 255, 0.9);`
    - `box-shadow: inset 0 0 0 1px rgba(37, 99, 235, 0.6);`
    - `background: rgba(37, 99, 235, 0.10);`
    - `position: absolute;` with top/left/width/height from drag calculation
  - On `mouseup` on `document`: finalize the box position. Switch overlay to `pointer-events: none` (deactivate draw mode). Set selection box and corner handles to `pointer-events: auto`. Show corner handles.
  - Minimum box size: 20x20 CSS pixels. If smaller, discard the draw attempt.

**Corner handles (per D-04, D-06):**
- Four handles at the four corners of the selection box.
- Each handle: `width: 12px; height: 12px; background: #FFFFFF; border: 2px solid #2563EB; box-shadow: 0 0 2px rgba(0,0,0,0.3); position: absolute;`
- Positioning: nw at `top: -6px; left: -6px;`, ne at `top: -6px; right: -6px;`, sw at `bottom: -6px; left: -6px;`, se at `bottom: -6px; right: -6px;`
- Cursor per handle: `nw-resize`, `ne-resize`, `sw-resize`, `se-resize`
- Each handle gets a `mousedown` listener that:
  - Sets `pointer-events: auto` on the overlay container
  - On `mousemove` on `document`: resize the selection box from the corresponding corner. The opposite corner stays fixed.
  - On `mouseup` on `document`: finalize resize, restore `pointer-events: none` on overlay container
  - Minimum box size during resize: 20x20px

**Box dragging (move):**
- When user `mousedown`s inside the selection box (not on a handle):
  - Cursor: `move`
  - `mousemove` on `document`: translate the box position
  - `mouseup` on `document`: finalize position
  - Clamp box position so it stays within the overlay container bounds

**Coordinate conversion (per RESEARCH.md Pattern 5):**
- `getVideoIntrinsicRect()` converts CSS pixel coordinates to video intrinsic pixel coordinates:
  ```javascript
  const rect = this.videoEl.getBoundingClientRect();
  const scaleX = this.videoEl.videoWidth / rect.width;
  const scaleY = this.videoEl.videoHeight / rect.height;
  return {
    x: this.selectionCSS.x * scaleX,
    y: this.selectionCSS.y * scaleY,
    width: this.selectionCSS.width * scaleX,
    height: this.selectionCSS.height * scaleY,
  };
  ```
  where `this.selectionCSS` stores {x, y, width, height} relative to the video element's top-left.

**YouTube SPA navigation cleanup (per RESEARCH.md Pitfall 5):**
- Listen for `yt-navigate-finish` event on `document`.
- On navigation: call `destroy()` to remove all injected DOM and event listeners.
- The content script will re-initialize the overlay on the new page if needed.

**Preventing event leaks:**
- Store all `document`-level event listeners in an array so `destroy()` can remove them all.
- Use `{ once: true }` on `mouseup` where appropriate.
