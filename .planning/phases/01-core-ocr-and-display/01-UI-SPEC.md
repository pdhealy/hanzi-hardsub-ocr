# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Core OCR and Display. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — plain HTML/CSS/JS Chrome extension |
| Icon library | none — text labels only in Phase 1 |
| Font | system-ui, -apple-system, sans-serif (OS default stack) |

**Rationale (source: CONTEXT.md):** Greenfield Chrome extension. No framework (React/Next.js/Vite), so shadcn does not apply. All UI is hand-authored HTML/CSS injected by content scripts or rendered in the popup.

---

## Surface Inventory

This extension has three distinct UI surfaces. Each has its own constraints.

| Surface | Rendered By | Constraints |
|---------|-------------|-------------|
| Popup (`popup.html`) | Extension popup window | Fixed ~360px wide, auto height, Chrome popup chrome |
| Selection overlay | Content script injected `div` | Absolute positioned over YouTube video player element |
| Side panel | Content script injected `div` | Fixed positioned right edge of viewport, overlays YouTube layout |

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, corner handle offset |
| sm | 8px | Compact button padding, header internal gaps |
| md | 16px | Default element spacing, popup section padding |
| lg | 24px | Side panel header height, section breaks |
| xl | 32px | Side panel top/bottom internal padding |
| 2xl | 48px | Not used in Phase 1 |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:
- Corner handles on selection box: 12px × 12px squares (nearest multiple of 4 that is large enough for mouse targeting)
- Drag handle (left edge of side panel): 4px wide × 100% height — touch surface is 8px wide (4px handle + 4px invisible hit area)
- Side panel default width: 300px (source: CONTEXT.md D-12)

---

## Typography

All text is rendered in the system font stack. Chinese character OCR output must render correctly in the system font — no custom font loading.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | OCR output text in side panel, popup body copy |
| Label | 12px | 600 | 1.4 | Button labels, header title, status indicators |
| Heading | 16px | 600 | 1.2 | Side panel header ("YouTube Chinese Reader"), popup section headings |
| Status | 13px | 400 | 1.4 | "Recognizing...", "No text recognized", error messages |

**Chinese character rendering note:** OCR output text at 14px body must be legible for CJK characters. System fonts (PingFang SC, Microsoft YaHei, Noto Sans CJK) handle this automatically via the OS font stack — no explicit font-family declaration for CJK is needed beyond `system-ui, sans-serif`.

## Color

### Popup and Side Panel

Dark-on-light scheme. YouTube's page is complex and colorful — the injected UI must have enough contrast to be readable against any background.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` | Popup background, side panel background surface |
| Secondary (30%) | `#F3F4F6` | Popup button area background, side panel header background |
| Accent (10%) | `#2563EB` (blue-600) | Primary CTA button fill, active state indicator dot |
| Destructive | `#DC2626` (red-600) | Not used in Phase 1 (no destructive actions in scope) |
| Text primary | `#111827` | All body and heading text |
| Text secondary | `#6B7280` | Status text, placeholder copy |
| Border | `#E5E7EB` | Panel border, popup section dividers |

Accent reserved for: **primary CTA button ("Recognize Text")** and **recognition active state indicator** only. Not applied to borders, labels, icons, or secondary buttons.

### Selection Box Overlay

The selection box renders directly over video content of unknown color. High-contrast values required.

| Role | Value | Usage |
|------|-------|-------|
| Box border | `rgba(255, 255, 255, 0.9)` solid 2px | Outer border of selection rectangle |
| Box border inner shadow | `rgba(37, 99, 235, 0.6)` | 1px inner box-shadow for blue tint visible on light video content |
| Box fill | `rgba(37, 99, 235, 0.10)` | Semi-transparent blue fill of selection area |
| Corner handles | `#FFFFFF` solid fill, `#2563EB` 2px border | 12×12px squares at four corners |
| Corner handle shadow | `rgba(0,0,0,0.3)` 2px blur | Ensures handles are visible on white video frames |

Source: CONTEXT.md D-06 — "semi-transparent blue/white border with white corner handles."

## Interaction States

### Popup Buttons

| State | Visual |
|-------|--------|
| Default | `#2563EB` fill, white label (primary); `#F3F4F6` fill, `#111827` label (secondary) |
| Hover | Primary: `#1D4ED8`. Secondary: `#E5E7EB` |
| Active/Pressed | Primary: `#1E40AF`. Secondary: `#D1D5DB` |
| Disabled | `#9CA3AF` fill, white label, `cursor: not-allowed` |
| Focus | 2px `#2563EB` outline, 2px offset — keyboard accessible |

### Recognition Active Indicator

A small colored dot in the popup header area.

| State | Color | Label |
|-------|-------|-------|
| Inactive / no box drawn | `#9CA3AF` (gray) | "Inactive" |
| Box drawn, ready | `#22C55E` (green-500) | "Ready" |
| Processing OCR | `#F59E0B` (amber-500), pulsing | "Recognizing..." |

Source: REQUIREMENTS.md — "clear visual feedback indicating whether recognition is active or inactive."

### Selection Box Cursor States

| Interaction Zone | Cursor |
|-----------------|--------|
| Drawing mode active (before first drag) | `crosshair` |
| Inside drawn box (drag to move) | `move` |
| Corner handle (drag to resize) | `nw-resize` / `ne-resize` / `sw-resize` / `se-resize` (matching corner) |
| Outside box (drawing mode off) | `default` |

Source: CONTEXT.md D-03, D-04.

## Component Inventory

Components to build in Phase 1 (plain HTML/CSS — no component library):

| Component | Surface | Description |
|-----------|---------|-------------|
| `popup.html` | Extension popup | Two buttons ("Draw Subtitle Area", "Recognize Text"), recognition status indicator, extension title |
| `SelectionOverlay` | Content script | Absolute-positioned div wrapping the video player. Renders selection rectangle, 4 corner handles, semi-transparent fill. Manages crosshair/draw mode. |
| `SidePanel` | Content script | Fixed right-side injected div. Header row (title + hide button), scrollable OCR output area, left-edge drag handle. Default width 300px, min 200px, max 600px. |
| `StatusIndicator` | Popup | Colored dot + label text showing Inactive / Ready / Recognizing state |
| `LoadingSpinner` | Side panel | CSS-only spinning ring shown during OCR processing. Replaces output area content. |

## Layout Specifications

### Popup Layout

```
┌────────────────────────────┐
│  YouTube Chinese Reader    │  ← heading 16px/600, padding 16px
│  ● Inactive                │  ← status indicator 12px/600
├────────────────────────────┤
│  [Draw Subtitle Area]      │  ← secondary button, full width, 8px padding
│  [Recognize Text]          │  ← primary button (accent), full width, 8px padding
└────────────────────────────┘
```

- Popup total width: 280px
- Button height: 36px (padding: 8px 16px)
- Gap between buttons: 8px
- Outer padding: 16px all sides

### Side Panel Layout

```
┌─────────────────────────────┐
│ YouTube Chinese Reader  [×] │  ← header 40px tall, bg #F3F4F6
├─────────────────────────────┤
│                             │
│   OCR output text area      │  ← scrollable, padding 16px
│   (or loading spinner)      │
│   (or empty/error state)    │
│                             │
└─────────────────────────────┘
 ↑ drag handle (4px, left edge)
```

- Panel position: `fixed`, `top: 0`, `right: 0`, `height: 100vh`
- Default width: 300px
- Min width: 200px, max width: 600px
- z-index: 9999 (above YouTube UI)
- Header height: 40px, `display: flex`, `align-items: center`, padding `0 16px`
- Close button: 24×24px, text "×", positioned `margin-left: auto`
- Output area: `overflow-y: auto`, `padding: 16px`, `flex: 1`

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard focus for popup buttons | `focus` outline: 2px `#2563EB` at 2px offset |
| ARIA labels for icon-only controls | Close button: `aria-label="Hide Panel"` |
| Loading state announcement | Side panel loading container: `aria-live="polite"` `aria-busy="true"` |
| OCR output announcement | Output container: `aria-live="polite"` — announces new text when OCR completes |
| Color contrast | All text/background combinations meet WCAG AA (4.5:1 for body, 3:1 for large text) |
| Selection box keyboard fallback | Not required in Phase 1 — mouse-only interaction is acceptable for draw mode |
