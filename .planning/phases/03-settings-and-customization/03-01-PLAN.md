# Phase 3 Plan 01: Settings Page and Style Customization

## Objective

Create a settings page (options page) for the extension with controls for font size, font color, and panel background opacity. Persist the settings using `chrome.storage.sync`. Apply the saved settings to the side panel text entries in real time.

## Context

The existing side panel (`extension/content/sidepanel.js`) renders OCR text entries with hardcoded CSS. The popup (`extension/popup/popup.html`) is the only current UI. The manifest already declares the `storage` permission.

Key files to understand before implementing:
- `extension/content/sidepanel.js` — SidePanel class, PANEL_STYLES constant, `.ycr-entry` and `.ycr-text` CSS classes
- `extension/popup/popup.css` — Design language: system-ui font, #111827 text, #2563EB primary, 6px border-radius
- `extension/manifest.json` — Already has `storage` permission; needs `options_page` added
- `extension/background/service-worker.js` — Background service worker for messaging

## Tasks

### Task 1: Options page HTML and CSS
**Type:** auto
**Goal:** Create `extension/options/options.html` and `extension/options/options.css` with a settings form containing three controls: font size (range slider 12-28px), font color (color picker), and panel background opacity (range slider 0.1-1.0).

**Implementation:**
- Create `extension/options/` directory
- `options.html`: Standalone HTML page (loaded in a browser tab, not as popup). Three labeled controls:
  1. Font Size: `<input type="range" id="font-size" min="12" max="28" step="1">` with a live readout `<span id="font-size-val">14px</span>`
  2. Font Color: `<input type="color" id="font-color">` defaulting to `#111827`
  3. Background Opacity: `<input type="range" id="bg-opacity" min="0.1" max="1" step="0.05">` with `<span id="bg-opacity-val">1.0</span>`
  4. Save button: `<button id="btn-save">Save Settings</button>`
  5. Status message: `<span id="save-status"></span>` (shows "Saved!" briefly after save)
- `options.css`: Consistent with popup design language (system-ui font, #111827 text, #2563EB primary color, 6px border-radius, white background). Layout: single column, max-width 480px, centered, card-style padding.
- No `<script src="...">` in options.html yet (added in Task 2).

**Done criteria:**
- `extension/options/options.html` exists and is valid HTML5
- `extension/options/options.css` exists with styles matching design language
- All three controls present with correct `id` attributes

### Task 2: Options page JavaScript with chrome.storage.sync
**Type:** auto
**Goal:** Create `extension/options/options.js` that loads saved settings from `chrome.storage.sync` on page open, updates live readouts on range input, and saves to `chrome.storage.sync` on Save button click.

**Storage schema:**
```json
{
  "ycrFontSize": 14,
  "ycrFontColor": "#111827",
  "ycrBgOpacity": 1.0
}
```

**Implementation:**
- `options.js`: Vanilla JS (no bundling required — options page is a standard browser page loaded in its own tab, can use `chrome.storage` API directly as it's an extension page)
- On DOM load: read from `chrome.storage.sync`, populate all three controls and their readout spans with saved values (or defaults if not set)
- Input event handlers for both range inputs: update the adjacent readout span in real time
- Save button click: write all three values to `chrome.storage.sync`, then briefly show "Saved!" in `#save-status` (fade out after 2 seconds)
- Defaults: fontSize=14, fontColor='#111827', bgOpacity=1.0
- Add `<script src="options.js"></script>` to options.html (no bundle needed)

**Done criteria:**
- `extension/options/options.js` exists
- `options.html` links to `options.js`
- Load/save/readout behavior implemented

### Task 3: Register options page in manifest and wire settings to side panel
**Type:** auto
**Goal:** Add `"options_page": "options/options.html"` to manifest.json. Update sidepanel.js to read settings from `chrome.storage.sync` and apply them dynamically to the side panel entries.

**Implementation in sidepanel.js:**
- Add a `_applySettings(settings)` private method that injects a `<style id="ycr-entry-styles">` element (separate from the existing `#ycr-panel-styles`) into `document.head` with CSS variables or direct rules:
  ```css
  .ycr-entry .ycr-text { font-size: {fontSize}px; color: {fontColor}; }
  #ycr-side-panel { background: rgba(255,255,255,{bgOpacity}); }
  ```
- Add a `loadSettings()` method that calls `chrome.storage.sync.get({ycrFontSize:14, ycrFontColor:'#111827', ycrBgOpacity:1.0}, cb)` and calls `_applySettings(settings)` with the result
- Call `this.loadSettings()` at the end of `_init()` so settings apply when the panel is first created
- Listen for `chrome.storage.onChanged` in `_init()` to re-apply settings when the user saves from the options page — call `_applySettings()` with the new values merged into current
- Add `"options_page": "options/options.html"` to `extension/manifest.json`
- Rebuild bundles: `npm run build`

**Done criteria:**
- `manifest.json` has `options_page` field
- `sidepanel.js` has `loadSettings()` and `_applySettings()` methods
- `_applySettings` injects `#ycr-entry-styles` style element with correct CSS
- Bundle rebuilt successfully
